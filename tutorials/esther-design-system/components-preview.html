1E-C. 手写批注

Key Insight ✦

用户不会读说明书

界面应该自我解释。如果需要说明书，说明设计还不够直觉化。把复杂留给系统，把简单留给用户。

3A. 极简留白式金句

在做任何设计决策之前，先问自己：这对用户有什么价值？如果答不上来，就不要做。

价值驱动的设计，永远比审美驱动的设计走得更远。

3B. 手写批注风

Key Insight ✦

组件库不是设计系统。设计系统包含原则、模式、语言、流程——组件只是其中最可见的产出物。

3C. 纯排版强调

THE DAILY BRIEF

设计系统的价值不在工具层

很多团队把设计系统等同于组件库，这是最常见的误区。真正的设计系统是一套共识——关于如何做决策、如何保持一致、如何高效协作的共识。

↑ 核心观点！

组件粒度

原子组件太细，页面模板太粗。找到恰当的抽象层次是设计系统成败的关键。

文档先行

没人用的设计系统不如没有。好的文档 = 低门槛 + 好搜索 + 真实案例。

重点！！

设计系统的意义

当一个产品有了统一的设计语言，团队里的每个人——无论是设计师、工程师还是产品经理——都能用同一套词汇沟通。组件不是束缚创造力的牢笼，而是让创造力能被快速、可靠地落地的基础设施。好的设计系统像语法规则：有了它，每个人的表达才能被准确理解。
