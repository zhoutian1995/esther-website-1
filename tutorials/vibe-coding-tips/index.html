02

让AI告诉你一个网站有哪些部分

"一个个人网站通常有哪些模块？"

直接问AI，它比你想得更了解结构

AI列出：Hero区、关于我、作品集、时间线、联系方式等

它会给你一个完整清单

你来选要哪些、不要哪些，定下结构

做减法，不是什么都要

先定骨架再填肉，不要一口气全丢给AI

分步走永远比一步到位质量高
Tip 03 · The Core

确定每个模块的内容

💬

逐个模块对齐——
"关于我这块写什么？""作品集放哪些？"

✍️

把你的真实信息填进去

🧠

尽可能投喂多的信息给AI——它不知道你是谁，你喂得越多，出来的东西越像你

❗

这个部分是最需要花时间的——因为这才是整个网站的核心。每个模块对应展示你的一部分能力、经验、背景，这些内容才是让别人记住你的东西

Tip 04 · Visual 04

让AI学习你的视觉风格

  • 丢你已有的网站/品牌色/喜欢的风格给它看
  • "参考这个网站的配色和排版风格"
  • 给参考比描述风格有效10倍
Show, don't tell ✨
Tip 05 · Workflow

分模块搓，每次只做一个部分

1

先做Hero区

让它一口气出5个方案

2

你选一个

挑最接近心意的，再优化

3

逐个搓完

再做"关于我"，同样的流程

4

不要一次写完

分块做质量高得多

Tip 06 · Assemble 06

组装+调细节

  • ✓ 把所有模块拼在一起
  • ✓ 调间距、调颜色、调动画
  • ✓ 手机端看一下
Tip 07 · Ship It

部署上线（GitHub Pages）

做完了不能只在本地看，得让全世界都能访问

  1. 打开 GitHub → 新建一个仓库（比如 my-website）
  2. 把你的 HTML 文件上传上去（直接拖进去就行）
  3. 进仓库的 Settings → Pages → Source 选 main 分支 → Save
  4. 等1分钟，你的网站就上线了 🎉
https://你的用户名.github.io/my-website

GitHub Pages 完全免费，不需要买服务器，一个HTML文件就是一个网站 ✌️
